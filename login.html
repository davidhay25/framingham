Demonstration of using OrionHealth API's for Analytics

This application is an example of how to access data from the Orion Health APIs, and to write data back. It calculates the Framingham risk score for patients in a providers list, and optionally writes the score back as a risk Observation.

There are a number of steps demonstrated

  1. First, you need to log in to the system. You need to have a Portal login to do this, and have access to clincial data. (Technically, this is done using OAuth2). When you click the 'Login' button below, a login form will be displayed
  2. Once you have logged in, the application will retrieve the details of the user, including the worklists that have been defined for that user.
  3. Next, you can select a list to examine. This will display the patients in the list, and will also calculate the Framingham score for all the patients in the list - based on the data in the system and accessed via the FHIR APIs. For your convenience, the first list is automatically retrieved and displayed.
  4. Finally, you can instruct the application to save the risk score back to the system.
